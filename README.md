# Бот сбора статистики Telegram-каналов

Заготовка на **aiogram (Bot API)** для сбора статистики со **своих** каналов,
где бот добавлен администратором.

## Что собирает

- **Подписчики** — снимок числа подписчиков по расписанию → динамика во времени (таблица `subscribers`).
- **Посты** — каждый новый пост канала логируется (таблица `posts`).

Данные пишутся в локальный SQLite-файл (`stats.db`).

## Ограничение

Bot API **не отдаёт просмотры постов**. Для просмотров/реакций нужен MTProto
(библиотека Telethon, `api_id`/`api_hash` с https://my.telegram.org). Точка
расширения помечена в `bot.py` как TODO.

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env       # заполните BOT_TOKEN и CHANNELS
```

### Шаги

1. Создайте бота через **@BotFather** → получите токен → впишите в `BOT_TOKEN`.
2. Добавьте бота **администратором** в каждый канал.
3. Укажите каналы в `CHANNELS` (через запятую: `@username` или id вида `-100...`).
4. Запустите:

```bash
python bot.py
```

## Как посмотреть собранные данные

```bash
sqlite3 stats.db "SELECT channel, members, taken_at FROM subscribers ORDER BY id DESC LIMIT 10;"
sqlite3 stats.db "SELECT channel, message_id, posted_at FROM posts ORDER BY id DESC LIMIT 10;"
```

## Просмотры, реакции, комментарии (MTProto / Telethon)

Bot API этого не отдаёт. Скрипт `collector_mtproto.py` берёт последние
`POSTS_LIMIT` постов каждого канала и пишет в таблицу `post_metrics`:
просмотры, пересылки, реакции (всего + разбивка), число комментариев.

Запуск:

```bash
python collector_mtproto.py           # разовый сбор
python collector_mtproto.py --loop    # повтор каждые POLL_MIN минут
```

Требуется:

1. `API_ID` и `API_HASH` с https://my.telegram.org → впишите в `.env`.
2. Первый запуск — вход по номеру телефона и коду (создаётся файл `.session`).
3. Для **приватных** каналов ваш аккаунт должен быть **участником** канала.
4. Комментарии считаются, только если у канала есть **группа обсуждений**.

### Если Telegram заблокирован в вашей сети

Telethon умеет работать через прокси — заполните в `.env` блок `PROXY_*`
(`socks5`/`socks4`/`http`). Тогда и сбор метрик, и первичный вход пойдут
через прокси.

## Настройки (.env)

| Переменная             | Назначение                                |
|------------------------|-------------------------------------------|
| `BOT_TOKEN`            | Токен от @BotFather                       |
| `CHANNELS`             | Каналы через запятую                      |
| `SNAPSHOT_INTERVAL_MIN`| Период снятия подписчиков (мин), деф. 60  |
| `DB_PATH`              | Путь к файлу БД, деф. `stats.db`          |
