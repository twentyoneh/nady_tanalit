# Подписки / отписки / источники через скрапинг (без api_id)

Полный путь от логина до выгрузки в Excel. **`API_ID`/`API_HASH` не нужны** —
всё через ваш веб-логин в web.telegram.org (`/a/`, клиент WebZ) + Playwright.

Две части:

| Что | Скрипт | Куда пишет | Надёжность |
|-----|--------|------------|------------|
| Подписчики по датам (рост/отток) | `web_scraper.py` | таблица `subscribers` | высокая |
| Разбивка новых подписчиков по источникам | `stats_scraper.py` | таблица `channel_stats` | нужен один прогон-разведка |

`stats.db` — служебное хранилище (копит историю, убирает дубли), вы его не
открываете. Работаете только с `stats_export.xlsx`.

---

## Весь путь одной шпаргалкой

```bash
# 1. Установка (один раз)
pip install -r requirements.txt
python3 -m playwright install chromium
cp .env.example .env          # вписать CHANNEL_NAMES (названия каналов)

# 2. Логин в Telegram Web (один раз, QR или номер+код)
python3 web_scraper.py --login

# 3. Подписчики (рост/отток) — регулярно или по расписанию
python3 web_scraper.py

# 4. Источники — сначала РАЗВЕДКА (один раз), потом сбор
python3 stats_scraper.py --recon
python3 stats_scraper.py

# 5. Выгрузка в Excel
python3 export_excel.py        # -> stats_export.xlsx
```

Ниже подробно.

---

## 1. Что нужно заранее

1. Вы **админ** отслеживаемых каналов (для раздела «Статистика» — обязательно).
2. У канала **открыт раздел «Статистика»** (канал дорос до порога подписчиков).
   Если раздела нет в приложении — источников не будет и у скрипта.
3. Установлен **Playwright** и браузер Chromium.

`API_ID`/`API_HASH` не нужны — вход по номеру/QR прямо в веб-версии.

---

## 2. Установка

```bash
pip install -r requirements.txt
python3 -m playwright install chromium
cp .env.example .env
```

В `.env` для скрапинга важны:

```ini
CHANNEL_NAMES=Мой канал,Второй канал   # названия каналов, как в Telegram
USER_DATA_DIR=tg_web_profile_a         # папка с сохранённым логином браузера
DB_PATH=stats.db
WEB_PROXY=                             # при VPN оставить пустым
```

---

## 3. Логин (один раз)

```bash
python3 web_scraper.py --login
```

Откроется окно браузера — войдите (QR или номер + код). Сессия сохранится в
папке `USER_DATA_DIR`, повторно входить не нужно. Этот же логин использует и
`stats_scraper.py`.

---

## 4. Подписчики: рост и отток по датам

```bash
python3 web_scraper.py
```

По каждому каналу из `CHANNEL_NAMES` берётся число подписчиков из шапки
(«N subscribers»), парсится и пишется снимком в таблицу `subscribers`
(и заодно собираются метрики постов). Запускайте регулярно — каждый запуск
это новая точка во времени; разницу между точками считает Excel-выгрузка.

По расписанию (пример cron, ежедневно в 03:00):

```cron
0 3 * * * cd /путь/к/проекту && /usr/bin/python3 web_scraper.py >> subs.log 2>&1
```

---

## 5. Источники: сначала разведка, потом сбор

Разбивка по источникам живёт в **canvas-графиках** «Статистики» — чисел в HTML
нет. `stats_scraper.py` перехватывает данные графиков на лету (хук `JSON.parse`).
Открыть «Статистику» автоматически в разных версиях верстки ненадёжно, поэтому
её открываете **вручную**, а скрипт снимает данные.

### Шаг 1 — разведка (один раз)

```bash
python3 stats_scraper.py --recon
```

Для каждого канала: скрипт откроет канал → вы вручную откроете «Статистику»,
дождётесь графиков и нажмёте Enter. Скрипт выгрузит перехваченное в
`stats_dump_<канал>.json` и разметку в `stats_dom_<канал>.html`. **В базу не пишет.**

Файлы `stats_dump_*.json` нужны, чтобы точно определить, какой график —
источники, какой — подписки/отписки (там есть нюанс «источники подписок» vs
«источники просмотров»). После настройки классификации — обычный сбор.

### Шаг 2 — сбор

```bash
python3 stats_scraper.py
```

То же, но перехваченные графики классифицируются и пишутся в `channel_stats`.
Повторные запуски обновляют/дополняют дни без дублей.

## 6. Выгрузка в Excel

```bash
python3 export_excel.py                 # -> stats_export.xlsx
python3 export_excel.py моё_имя.xlsx     # своё имя файла
```

Листы (создаются только при наличии данных):

| Лист | Источник | Что внутри |
|------|----------|------------|
| **Подписчики** | `subscribers` | снимки числа подписчиков по времени |
| **Динамика** | `subscribers` | прирост между снимками (рост/отток) |
| **Подписки-отписки** | `channel_stats` | по дням: подписки, отписки, чистый прирост |
| **Источники подписок** | `channel_stats` | сводная день × источник + «Всего» |
| **Рост канала** | `channel_stats` | размер канала по дням |
| **Метрики постов** | `post_metrics` | просмотры/реакции/комментарии постов |

Несколько каналов идут в тех же листах, разделены колонкой «Канал».

---

## 7. Если что-то не так

| Симптом | Причина / решение |
|---------|-------------------|
| `Приложение не загрузилось` | Не залогинены — `python3 web_scraper.py --login`. |
| `нет видимых результатов поиска` | Имя в `CHANNEL_NAMES` не совпало — впишите как в Telegram. |
| `перехвачено графиков: 0` | «Статистика» не открылась/не дорисовалась. Откройте её и дайте графикам прогрузиться до Enter. |
| Пусто на листе «Источники» | Канал ещё не отдаёт статистику (мало подписчиков) или вы не админ. |
| Не подключается / таймауты | Заблокирован Telegram — включите VPN или заполните `WEB_PROXY`. |
| `ValueError: invalid literal for int()` от api_id | Вы запустили старый Telethon-скрипт. В скрапинговом пути он не нужен. |

---

## 8. Что НЕ используется в этом пути

`bot.py` (Bot API) и `collector_mtproto.py` (Telethon) — старая ветка, ей
нужен `api_id`. Для скрапинга они не нужны и не запускаются.
